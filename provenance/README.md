# provenance

This folder holds the source material from which the ideas in this repo were developed: conversation transcripts, working notes, earlier research, and other artifacts of the thinking process.

It exists because this project is, in part, about itself. The core claim of *throughlines* is that the durable layer of a working life is the record of how your thinking and systems unfolded — the moves you made, the reasons behind them, what you tried and kept and backed off. This folder applies that claim to the repo itself. The polished content in the rest of the repo is one output. The traceable record of how that content came to be is another, and it's kept here.

## What's in here

- **transcripts/** — conversations (with LLMs, with collaborators, with yourself via voice memo) that produced or shaped the ideas in this repo. Raw-ish: lightly edited for readability, but not polished. Filenames use `YYYY-MM-DD-short-slug.md`.

- **prior-work/** — earlier writing, notes, and research repos that have been consolidated into this project. Each subfolder contains an `IMPORT-NOTE.md` explaining when it was imported, where it came from, and what it contains. Files inside are preserved as they were at import; they use their original vocabulary and framing, which may not match the current kit.

- **notes/** — working notes, half-formed thoughts, drafts that didn't get promoted, reading reactions. Less structured than transcripts or prior-work. Use freely; prune rarely.

Other subfolders may appear as the project accumulates different kinds of source material (correspondence, whiteboard photos, meeting notes, etc.). Each subfolder gets its own short README if the convention for what goes there isn't obvious.

## How to read this folder

If you're here to understand the finished ideas, start with the root README and the `characteristics/` folder — those are the polished outputs.

If you're here because you want to see how the ideas actually emerged — the false starts, the naming detours, the earlier research that seeded later framings, the places where an offhand comment reshaped the concept — this is the right folder. Expect unevenness. Expect the thinking to be better in some places than others. Expect some entries to have been overtaken by later revisions; that's part of the point.

For prior-work entries specifically: expect vocabulary and framing that predates the current project. An earlier file may use terms that have since been renamed, or argue for distinctions that have since been collapsed. That's not a problem to fix — it's a record of what was thought at the time.

## Why keep this at all

Three reasons.

**First, honesty.** Finished work tends to present itself as if it arrived fully-formed. It never does. Keeping the provenance visible is a small correction to that distortion, for readers and for the author.

**Second, regeneration.** If the repo's polished layer is ever lost, damaged, or needs rewriting, the provenance is the input that makes regeneration possible. The durable layer isn't the conclusions; it's the record that produced them.

**Third, dogfooding.** The project argues that individuals should keep a durable record of their own unfoldings. Keeping one for this repo is the minimum required to take the idea seriously.

## A note on editing

Provenance entries are not edited for accuracy of the ideas they contain. An entry from six months ago may contain claims the author no longer believes or vocabulary that's since been revised. An imported prior-work file may use terms that the current project has moved past. That's expected and wanted. Provenance is a record of what was thought *at the time*, not a running summary of current belief. When the current view differs substantially, that belongs in the polished content elsewhere in the repo — not in revisions to the provenance itself.

Light editing for readability (fixing obvious typos, adding headers, redacting private information) is fine. Rewriting for correctness is not.
